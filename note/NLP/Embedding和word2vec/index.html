# Embedding

# 概述

Embedding一词广泛出现在自然语言理解、推荐系统、CV等方向。本意是“嵌入式，嵌入层”。 通过Embedding，可以把现实世界中的文字、图片、语言、视频等转化为计算机能识别、能使用的语言，并且转化过程中信息不丢失。

Embedding是用一个低维的向量表示一个物体（词，商品，可以是万物），Embedding向量的性质使距离相近的向量对应的物体有相近的含义。

比如 Embedding(复仇者联盟)和Embedding(钢铁侠)之间的距离就会很接近，但 Embedding(复仇者联盟)和Embedding(乱世佳人)的距离就会远一些。

Embedding也具有数学运算关系，如Embedding（马德里）-Embedding（西班牙）+Embedding(法国)≈Embedding(巴黎)。

Embedding和傅里叶变换有共通之处：从另一个空间表达物体，甚至揭示物体间的潜在关系。

Embedding相对于one-hot编码的优势：把物体编码到一个低维稠密向量（并且还能保留其含义 ）喂给DNN，使得更高效。one-hot encoding是一种稀疏向量的编码方式。

# 深度学习不利于稀疏特征向量处理

深度学习的结构特点不利于稀疏特征向量的处理：

①神经网络的梯度下降学习过程中，稀疏特征会导致整个网络收敛非常慢，每一个样本的学习只有极少数的权重会更新，在样本数量有限的情况下导致模型不收敛。

②one-hot稀疏特征的维度往往非常大，这样会导致网络的参数量非常庞大，导致很大的算力开销。

但one-hot编码把上面变成：

即：把每一个字都对应成一个十个（样本总数/字总数)元素的数组/列表，其中每一个字都用唯一对应的数组/列表对应，数组/列表的唯一性用1表示。

one-hot encoding的维度 = 物体的总数。

对于每一个特征，如果有m个可能值，经过one-hot编码后，变成了m个二元特征，这些特征互斥，每次只有一个激活，数据变稀疏。

这样做使得计算简单。稀疏矩阵做矩阵计算时，只需要把1对应的位置的数相乘求和即可。

但有明显缺点：过于稀疏，占用过多资源。

# Embedding层

Embedding层，某种程度上来说，就是用于降维的，本质是矩阵乘法。

一个m×dm \times dm×d的矩阵乘以d×nd \times nd×n的矩阵，最终得到一个m×nm \times nm×n的矩阵。

此外，Embedding层可以进行升维。低维空间内可能特征非常“笼统”，不便于观察，需要升到高维空间，把其中一些特征进行放大或者把笼统的特征清晰化。

# 语义理解中的Embedding

通过将两个无法比较的文字映射成向量，实现计算。

如：

queen = king -man +woman：即皇后是女性的国王。

walked = walking -swimming +swam：即walked是walking的过去式。

另外，向量间的距离也可能会建立联系。

如：|中国|-|北京| = |法国|-|巴黎|

通过嵌入的方式，实现两个世界的沟通。

例如：

用向量方式从智力层面描述小明。已知：小明的语文成绩88，数学成绩3，英语成绩18，身高149，体重35公斤，父亲是大学教授，母亲是音乐家，立定跳远2.1米，50米自由泳个人记录93秒。

根据已有信息（特征），小明的完整向量表示为：[88, 3, 18, 149, 35, 大学教授, 音乐家, 2.1, 93]。

根据先验知识，身高、体重、立定跳远、游泳成绩、家世和智力无明显关系，摒弃掉，仅保留[语文成绩，数学成绩，英语成绩] 三个维度的特征。

得小明的智力向量: [88, 3, 18]。

  1. Embedding的数据维度压缩（9维压缩到3维度）
  2. Embedding通常是丢失信息的——冗余信息和部分有用信息丢失

# 关于Embedding的一些观点

# Embedding本质是one-hot为输入的单层全连接

Embedding的数学本质，是以one-hot为输入的单层全连接。

世界上本没什么Embedding，有的只是one-hot From:https://www.zhihu.com/question/374835153 (opens new window)

Embedding和one-hot的不同，本质而言是矩阵乘法的实现。

# Embedding分布式表示方法

邱锡鹏老师的《神经网络与深度学习》一书。Embedding是一种分布式表示方法，即把原始输入数据分布地表示成一系列特征的线性组合。以RGB颜色为例：

只需要一个三维的稠密向量就可以表示所有的颜色，并且分布式表示也很容易表示新的颜色。

具体到NLP任务中，词的embedding，每一个句子都被表示成指定维度（如768）的向量，每一个维度对应句子中一个语义特征。

在NLP中，无法从语言学角度先验直到每一个维度（embedding矩阵的列向量）表示哪一种语义特征，也不知道token对应的特征值（token对应句子中的一个词，embedding矩阵的行向量），需要语言模型训练得到。word2vec和BERT也都是设计任务在海量语料上自监督学习得到对应的特征值。

Google的word2vec使得embedding方法流行。word2vec是实现CBOW（Continuous Bag-of-Words）和Skip-gram语言模型的工具。

Skip-gram的动机：每个词都决定了相邻的词。

# Skip-gram with Negative Sampling

上面模型执行三个步骤并输出预测向量，模型未训练，预测结果是错误的。

针对输入句子：Thou shalt not make a machine in the likeness of a human mind.

输入单词（not）的目标单词是thou，其概率为1，其它所有单词概率为0，这样组成了“目标向量”。将模型预测（model prediction）和目标向量（actual target）相减得到偏差向量。

偏差向量被用于模型更新：

这样下去遍历所有的样本（一个epoch），几轮epoch之后，得到训练好的模型，从中提取嵌入矩阵（embedding matrix）用于下游任务。

上面涉及到负采样（negative sampling)的概念：

在数据集中随机引入负样本——不是邻居的单词样本，并且这些样本的概率值为0。这些样本是从vocab词表中随机抽取，模拟噪声对比估计。

# word2vec训练与embedding矩阵

训练开始阶段，创建上述两个矩阵：

Embedding矩阵中嵌入了词表（vocab）中的每个单词，维度之一是vocab_size，第二个维度是每一次嵌入的长度（embedding_size），这个是设定值。【Embedding矩阵作为网络的Embedding层参数，跟随网络一起学习训练。】

Context矩阵是放入了词表中的每个单词。

上图展示：输入单词（not）和输出的上下文单词：thou（实际邻居词）、aaron和taco（引入的负样本)。

  1. 对于输入词，查找embedding矩阵中对应的向量
  2. 对于上下文词，查找Context矩阵对应的向量
  3. 计算输入和每个上下文嵌入向量的点积【余弦相似度】。并使用sigmoid激活函数，得到0-1之间的概率值。计算出误差error。
  4. 利用误差分数来调整not、thou、aaron和taco的嵌入。
  5. 多次迭代epoch训练后，Embedding嵌入矩阵得到改进，丢弃Context矩阵，使用Embedding矩阵用于下游任务

上述训练有两个关键的超参数：滑动窗口大小和负样本的数量。

窗口大小选择的一个启发方式：

较小窗口（2-15）得到：高相似得分表明两个词可互换（距离近）。

较大窗口（15-50）得到：相似性更能指示单词相关性的嵌入。

Gnesim默认窗口大小为5，默认的负采样样本数为5，当数据集足够大时，2-5个负采样样本数也能达到理想效果。
