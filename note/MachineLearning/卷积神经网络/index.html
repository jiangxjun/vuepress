# 发展历程

1998年 Yann LeCun教授在论文Gradient-Based Learning Applied to Document Recognition中提出了LeNet-5，将Backpropagation算法应用到神经网络结构训练中，形成了卷积神经网络（CNN）的雏形。

2012年，在ImageNet图像识别大赛中，Hinton组的论文ImageNet Classification with Deep Convolutional Neural Networks中提到了AlexNet，引入了全新的深层结构和Dropout正则化方法，CNN开始蓬勃发展。

# 基本概念

卷积核（滤波器）：filter。如3×3、5×5之类的。

特征图：feature map。输入和卷积核对应相乘累加计算得到的结果所组成，一个卷积核对应一个feature map。

卷积层：Convolutional layer。用于提取底层特征

池化层：pooling layer。防止过拟合，并降维

全连接层：fully-connected layer。汇总前面卷积层和池化层后特征，并输出结果，输出包括：softmax（各个分类的归一化概率分布）、sigmoid（二分类）

# 基本结构

大体上的基本结构主要包括：卷积层、池化层、全连接层

# 卷积计算

左边动的红色部分是一个不变的3×3的kernel，卷积核。滑动过程中，卷积核对应位置和Image对应位置进行乘积，并求和，得到右边输出结果。

卷积核光顾到的地方称为感受野（Receptive Field）。

提示

感受野（Receptive Field）：CNN每一层输出的特征图（feature map）上的像素点映射回输入图像上的区域大小。即特征图上一点相对于原图的大小，是CNN特征所能看到输入图像的区域。

感受野作用：

（1）小尺寸的卷积核代替大尺寸的卷积核，可以减少网络参数、增加网络深度、扩大感受野（如3个3×3的卷积核的叠加可以代替7×7的卷积核），网络深度越深，感受野性能越好。

（2）对于分类任务，最后一层feature map感受野大小要≥输入图像大小，否则分类性能不理想。

（3）对于目标检测任务，若感受野小，而目标尺寸大；或感受野大，目标尺寸小，模型收敛困难。

上图中，左边是原始输入数据，中间是卷积核，右边是输出的新的二维数据，称为特征图（feature map），

在CNN中，filter（带一组固定权重的神经元）对局部输入数据进行卷积计算，每计算完一个数据窗口内的局部数据后，数据窗口不断平滑移动，直到计算完所有数据，过程中有如下几个参数：

提示

①深度depth：神经元个数，决定输出的depth厚度，也代表滤波器filter个数，也是生成的feature map的个数

②步长stride：决定滑动多少步到边缘。【有时候需要stride来压缩一部分信息】

③填充padding：在外围边缘补充若干圈0，方便从初始位置以步长为单位可以刚好滑到末尾位置，即为了总长能被步长整除。使用padding是为了防止边缘特征被忽略。【有时候需要padding来避免信息损失】

常用的两种padding方式：

  • valid padding：不进行处理，只使用原始尺寸，并且不允许卷积核超出原始尺寸边界。
  • same padding：进行填充处理，允许卷积核超出原始尺寸边界，并且使得卷积后结果的尺寸和原始输入尺寸一致。

卷积计算：卷积核和在输入图像上对应滑动到的位置的像素相乘，并求和。

假设输入图片的尺寸：n×nn \times nn×n

卷积核大小：f×ff \times ff×f

填充padding大小：ppp

步长stride大小：sss

输出图片的尺寸：ooo

则输出尺寸的计算公式为：

o=⌊n+2p−fs⌋+1\begin{equation} o = \lfloor \frac{n+2p-f}{s} \rfloor +1 \end{equation} o=⌊sn+2p−f​⌋+1​​

其中，⌊⌋\lfloor \rfloor⌊⌋表示向下取整，用于结果不是整数时进行向下取整。

# 多通道卷积

一个可视化例子：https://thomelane.github.io/convolutions/2DConvRGB.html

卷积核红绿蓝部分，分别和输入的红绿蓝层进行做卷积计算，该“三阶魔方”所有结果相加，得到最后的输出。

该部分具体讲解参见：https://www.bilibili.com/video/BV1AJ411Q72b?p=3&spm_id_from=pageDriver

  • 左边7×7×3代表图像像素的长宽 ，3是RGB三个颜色通道
  • 中间部分是两个不同的滤波器，是3×3×3的卷积核
  • 两个神经元，depth=2，有两个滤波器
  • 数据窗口每次移动2个步长取3×3的局部数据，stride=2
  • zero-padding=1
  • 还有个偏置项bias，往往是一个标量，直接相加

分别以两个滤波器filter为轴滑动数组进行卷积计算，得到两组不同的结果。

假设stride=1，padding=0，如下图所示：卷积是在提取输入的底层特征。

上图中，3∗3∗33*3*33∗3∗3的卷积核和3通道的输入，对应维度进行卷积运算，最后再将三个维度的对应值相加，最后得到一个4∗44*44∗4的矩阵，加上偏置项，可以得到一个最终的单通道的输出。

若存在多个卷积核，把每一个卷积核对应的单通道输出组合成一个总的输出。

filter的通道数和输出的feature map的通道数

  • 某一层滤波器的通道数=上一层feature map的通道数，如上图6∗6∗36*6*36∗6∗3的RGB有3和通道，那么滤波器的也要有3个通道，即3个3∗33*33∗3的矩阵组成一个filter。
  • 某一层feature map的通道数=当前层滤波器的个数，如上图只有一个filter，输出feature map为4∗44*44∗4，只有1个通道；当有2个filter的时候，输出feature map应该是4∗4∗24*4*24∗4∗2，通道数为2。

提示

CNN中的“filter”是“卷积核”的一个集合，在只有一个通道下，卷积核相当于filter，当有多个通道时，filter就包括多个通道里面的卷积核。

真实情况中，卷积核不需要人工设置，而是通过机器学习方法（反向传播，梯度下降方法）学习得到的。

卷积核权重初始值设定，在Keras中为例：kernel_initializer = 'glorot_uniform'

# 池化层

原始图片是20×20的，对其进行下采样，采样窗口是10×10，最终将其下采样成为一个2×2的特征图。这样大大减少运算量，还可以有效的避免过拟合。

  • 最大池化（Max Pooling）：取区域的最大值。用的最多，
  • 平均池化（Average Pooling）：取区域的平均值

提示

使用max pooling，舍弃其他值，是否会造成信息损失问题：

  1. CNN中每一个卷积核可以看做一个特征提取器，不同的卷积核负责提取不同的特征，对其进行Max Pooling操作后，提取出的是真正能够识别特征的数值，其余被舍弃的数值，对于所要提取特定的特征并没有特别大的帮助。那么在进行后续计算使，减小了feature map的尺寸，从而减少参数，达到减小计算量。

  2. 并不是所有情况Max Pooling的效果都很好，有时候有些周边信息也会对某个特定特征的识别产生一定效果，那么这个时候舍弃这部分“不重要”的信息，就不划算了。具体情况具体分析，可以进行一个加与不加的结果输出对比，看最大池化是否有反效果。

池化的作用：

  1. 减少参数量，还保留原始数据特征
  2. 防止过拟合
  3. 为卷积网络带来平移不变性

# 全连接层

全连接层：输出的每一个神经元都和上一层的每一个神经元连接。

提示

卷积核(filter)数量、大小、移动步长（stride）、补0圈数（padding）、全连接层和隐含层层数、神经元个数 都是超参数（hyperparameters），超参数是由人为根据经验指定的。

网络内部权重参数等是由训练得到。

通过不断的设计卷积核的尺寸，数量，提取更多的特征，最后识别不同类别的物体。做完Max Pooling后，我们就会把这些数据“拍平”，丢到Flatten层，然后把Flatten层的output放到full connected Layer里，采用softmax对其进行分类。

​ 计算神经网络的预测结果与图片真实标签的差距，构造损失函数，训练的目标就是将找到损失函数的最小值，最后通过softmax给出预测分类的归一化概率分布决策。
