GPU加速必须在cuda9以上才可以完成加速，cuda9以后的版本在Linux系统上只支持Ubuntu16.04和Ubuntu17.04。

# 一、设备要求

系统：Ubuntu16.04及以上

显卡驱动：NVIDIA系列（384以上）

驱动检查：终端命令nvidia-smi

新安装的Ubuntu系统安装NVIDIA驱动步骤:

a.删除旧驱动：

`sudo apt-get remove --purge nvidia*

b.禁用自带的nouveau nvidia驱动：

看下Nouveau是否已经被禁用

lsmod | grep nouveau
1

如果已经没有任何显示说明不用禁用了，否则继续下面操作

sudo vim /etc/modprobe.d/blacklist-nouveau.conf #创建一个文件（注：按一下i键，表示现在进行内容插入）
1

并添加如下内容：

blacklist nouveau
options nouveau modeset=0
1

# 二、安装cuda9.0

官网下载 (opens new window)。有对应的deb和.run文件，有具体的安装命令。

推荐.run文件，因为deb文件安装，可能会修改已安装好的驱动和。

进入下载目录，给文件添加运行权限：

chmod +x ./cuda_9.0.176_384.81_linux.run
1

运行安装

sudo ./cuda_9.0.176_384.81_linux.run
1

启动安装程序，一直按空格到最后（可以选择Ctrl+c跳过），不用担心，到99%的时候，输入accept接受条款

注意：

  • 第一个提醒你是否安装驱动时，选“n”，其余都“y”，安装路径选择默认的路径。
  • 如果需要安装多版本CUDA，在安装到选择symtomic link to /urs/local/cuda时候，选择'n'，其它都是"y"。

以上为两种cuda的安装方法，不过并没有结束，安装完毕后必须添加环境

gedit ~/.bashrc
1
# 在文件末尾添加环境变量
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/cuda-9.0/lib64
export PATH=$PATH:/usr/local/cuda-9.0/bin
export CUDA_HOME=$CUDA_HOME:/usr/local/cuda-9.0
1
2
3
4

添加完后，终端命令：source ～/.bashrc

终端命令查看是否安装成功：nvcc -V，若出现nvcc未安装，

执行了 sudo apt-get install nvidia-cuda-toolkit 命令，nvcc命令可以正常执行，但是运行nvcc --version发现版本为5.5，与原来安装的CUDA 10.0不匹配，这将会导致一系列不兼容问题（比如在python中使用sk-cuda库就报错，因为nvcc版本不一致），遂通过如下步骤解决：

1、使用sudo apt-get autoremove nvidia-cuda-toolkit 卸载5.5版本

2、查看/usr/local/cuda/bin下是否有nvcc可执行程序，如果没有说明cuda没有正常安装，需要重新安装。

3、如果有nvcc可执行程序，切换到管理员权限，重新执行上面步骤，添加环境。

# 三、安装cudnn7.3.1

去官网下载 (opens new window)，此处下载的是cuDNN v7.3.1 Library for Linux版本。是一个.tgz文件。

下载完直接解压，解压会出现一个cuda文件夹，里面有两个文件include 和lib64，把里面的文件copy到/usr/local/cuda-9.0里面相应的目录里，具体命令如下：

sudo cp cuda/include/cudnn.h /usr/local/cuda-9.0/include
sudo cp cuda/lib64/libcudnn* /usr/local/cuda-9.0/lib64
sudo chmod a+r /usr/local/cuda-9.0/include/cudnn.h /usr/local/cuda-9.0/lib64/libcudnn*
1
2
3

查看cudnn版本，检查是否安装好：

cat /usr/local/cuda/include/cudnn.h | grep CUDNN_MAJOR -A 2
1

# 四、卸载

（1）卸载CUDA

sudo /usr/local/cuda-9.0/bin/uninstall_cuda_9.0.pl
1

（2）卸载CUDNN

删除原来的cudnn文件：

sudo rm -rf /usr/local/cuda-9.0/lib64/libcudnn*
sudo rm -rf /usr/local/cuda-9.0/include/cudnn.h
1
2

conda create -n pytorch_gpu python=3.7
conda activate pytorch_gpu /source activate pytorch_gpu #激动虚拟环境

二、安装pytorch GPU版本

此处是官网直接下载安装，具体考虑下载速度，可离线安装

##linux
conda install pytorch=1.7.1 torchvision torchaudio cudatoolkit=9.2 -c https://mirrors.bfsu.edu.cn/anaconda/cloud/pytorch/linux-64/

# 问题

nvcc –V查看的cuda版本不是软链接的cuda对应的版本问题解决

修改/etc/profile，在末尾加上：

vim /etc/profile

export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64$LD_LIBRARY_PATH
export CUDA_HOME=/usr/local/cuda

source /etc/profile

# 离线安装

但是有时候更换国内的源，下载仍会很慢，会中断，考虑去镜像源网站直接下载，然后离线安装，方法如下：

此处以ubuntu16.04系统下为例：

下载安装包：

pytorch-1.7.1-py3.7_cuda9.2.148_cudnn7.6.3_0.tar.bz2
torchaudio-0.7.2-py37.tar.bz2
torchvision-0.8.2-py37_cu92.tar.bz2
1
2
3

激活创建好的虚拟环境（pytorch），在该虚拟环境下，切换到下载文件的位置，执行命令安装：

(pytorch) $ conda install --offline pytorch-1.7.1-py3.7_cuda9.2.148_cudnn7.6.3_0.tar.bz2
(pytorch) $ conda install --offline torchaudio-0.7.2-py37.tar.bz2
(pytorch) $ conda install --offline torchvision-0.8.2-py37_cu92.tar.bz2
1
2
3

下载对应版本的pytorch、torchvision、cudatoolkit等，然后离线安装。

离线安装注意切换到对应的虚拟环境下进行，cd命令把当前目录改到下载的离线文件所在目录。

最好离线安装安装包的文件，对于一些依赖包，可以先在线安装好，最后离线安装如pytorch、torchvision、cudatoolkit等下载很慢的安装包。
