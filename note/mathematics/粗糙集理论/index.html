# 信息系统（Information system）

In Rough Set, data model information is stored in a table. Each row(tuples)(行/元组)represents a fact or an object. Often the facts are not consistent(符合，一致) with each other. In Rough Set terminology(术语), a data table is called an information system(IS). Thus, the information table represents input data, gathered from any domain.

Note: Rows of a table are called examples(objects, entities(实体)).

Formally, an information system(IS) is pair(U,AT)(U,AT)(U,AT), and UUU is a non-empty finite(有限的) set of objects, called the universe(论域), and ATATAT is a non-empty finite set of attributes(属性). If some attribute values are unknown, the IS is IIS. In IIS, the unknown attribute value is denoted by(用...表示) "*".

# 近似（approximations）

It is a formal approximation of a crisp set(明确的集合) defined by its two approximations- Upper approximation and Lower approximation.

# The rough set（粗糙集）

The tuple ⟨AˉX,A‾X⟩\langle \bar{A}X ,\underline{A}X \rangle⟨AˉX,A​X⟩ composed of the lower and upper approximation is called a rough set.

A set is said to be rough if its boundary region is non-empty, otherwise the set is crisp set(明确集).

The notion(概念) of Rough sets was introduced by Z Pawlak in his seminal paper of 1982[1]. It is a formal theory derived from fundamental research on the logical properties of information systems. Rough set theory has been a methodology(方法论) for database mining or knowledge discovery in relational databases. We can use a rough set approach to discover structural relationships within imprecise and noisy data[2].

Basic problems in data analysis solved by Rough Set:

  • Characterization(表征，描述) of a set of objects in terms of (在...方面)attribute values.
  • Finding dependency(依赖关系) between the attributes.
  • Reduction of superfluous(多余的) attributes.
  • Finding the most significant attributes.
  • Decision rule generation(生成决策规则).

Goals of Rough Set Theory:

  • The main goal of the rough set analysis is the induction(归纳，导致，产生) of (learning) approximations(近似) of concepts. Rough sets constitute(构成，组成) a sound(合理可靠的) basis of KDD(知识发现-Knowledge Discovery in Database). It offers mathematical tools to discover patterns hidden in data.

  • It can be used for feature selection, feature extraction, data reduction(数据简化), decision rule generation, and pattern extraction(templates, association rules(关联规则)) etc.

  • Identifies partial(部分) or total dependencies in data, eliminates(消除) redundant(冗余的) data, and gives an approach to null values, missing data, dynamic data and others.

# 知识发现（knowledge discovery in database）

KDD is the process of discovering useful knowledge from a collection of data. This widely used data mining technique is a process that includes data preparation and selection, data cleansing(数据清洗), incorporating(整合) prior knowledge of data sets and interpreting accurate solutions from the observed results(根据观测结果准确解释).

The KDD process has reached its peak(顶峰) in the last 10 years. It now houses(贮藏) many different approaches to discovery, which include inductive learning(归纳学习), Bayesian statistics, semantic query optimization(语义查询优化), knowledge acquisition for expert systems and information theory. The ultimate(最终) goal is to extract high-level knowledge from low-level data.

high-level describes those operations that are more abstract in nature

low-level describes more specific individual components of a systematic operation.

# 不完备信息系统（IIS）

In an incomplete information system (IIS), some attribute values for objects may be unknown (missing, null). But in practice, because of errors in data measuring, the limitations of acquiring data, some human factors, etc. IIS often occur in knowledge acquisition.

There are usually two methods in rough set theory（粗糙集理论） to deal with IIS: (1) data reparation（数据修复）and (2) model extension[3]. The model extension（模型扩展） is more suitable than data reparation for processing IIS, which does not change the original information of IIS and is more objective for the knowledge system generation.

# 广义不可分辨关系（generalized indiscernibility relations）

At present, various extension models of the classical rough set theory have been proposed, in which the indiscernibility relation was extended to some non-equivalence relations(非等价关系), or called generalized indiscernibility relations, such as tolerance relation(容差关系)[4], non-symmetric similarity relation(非对称相似关系)[5], limited tolerance relation(有限容差关系)[6], valued tolerance relation(量化容差关系)[7], and characteristic relation(特征关系)[8].

These extension models have their own limitations[3:1].

In tolerance relation, two objects that have no one known and the same value on all attributes may be considered as indiscernible(不可分的) and classified in the same class.(适合处理未知属性均为遗漏的)

In non-symmetric similarity relation, two objects that have a lot of known and same values on all attributes may be separated, and classified into different classes.(适合处理未知属性均为丢失的)

The limited tolerance relation inherits(继承) the merit(优点) of the tolerance relation and non-symmetric similarity relation and avoids their limitations. But, in limited tolerance relation, two objects may be considered similar if only they have one known and the same value on all attributes. This condition is too lenient(宽泛的，不严谨的) for the IIS with a large number of attributes.(针对具有大量属性的不完备信息系统而言，限制性容差关系条件过于宽泛)

The characteristic relation, which is a generalization of both tolerance relation and non-symmetric similarity relation, can not avoid their limitations.

# 特征关系（characteristic relation）

We will consider four kinds of missing attribute values, called lost(丢失型), "do not care" conditions(不关系条件型), restricted "do not care" conditions(限制不关心条件型) and attribute-concept values(属性-概念值)[8:1].

We will assume that lost values will be denoted by "?", "do not care" conditions by "*", restricted "do not care" conditions by "+", and attribute-concept values by "-".

An important tool to analyze complete decision tables is a block of the attribute-value pair. Let aaa be an attribute, and a∈Aa \in Aa∈A. Let vvv be a value of aaa for some cases.

For complete decision tables if t=(a,v)t= (a,v)t=(a,v) is an attribute-value pair, then a block of ttt, denoted [t][t][t], is a set of all cases from UUU that for attribute aaa has value vvv.

对不完备信息系统而言，只有已知属性值才有属性值对块，未知属性值没有属性值对块。

Additionally, any decision table defines a function ρ\rhoρ that maps the direct product(乘积) of UUU and AAA into the set of all values.

  • If for an attribute aaa there exists a case xxx such that ρ(x,a)=?\rho(x,a) = ?ρ(x,a)=?, i.e., the corresponding value is lost, then the case xxx should not be included in any blocks [(a,v)][(a,v)][(a,v)] for all values vvv of attribute aaa.

    丢失型，该case不包括在任何的属性值对块中的。

  • If for an attribute aaa there exists a case xxx such that the corresponding value is a "do not care" condition or a restricted "do not care" condition, i.e., ρ(x,a)=∗\rho(x,a) = *ρ(x,a)=∗ or ρ(x,a)=+\rho(x,a) = +ρ(x,a)=+, then the case xxx should be included in blocks [(a,v)][(a,v)][(a,v)] for all specified values vvv of attribute aaa.

  • If for an attribute aaa there exists a case xxx such that the corresponding value is an attribute-concept value, i.e., ρ(x,a)=−\rho(x,a) = -ρ(x,a)=−, then the corresponding case xxx should be included in blocks [(a,v)][(a,v)][(a,v)] for all specified values v∈V(x,a)v \in V(x,a)v∈V(x,a) of attribute aaa, where:

V(x,a)={ρ(y,a)∣ρ(y,a)isspecified,y∈U,ρ(y,d)=ρ(x,d)}.V(x,a) = \{\rho(y,a) \ | \rho(y,a) is \ specified, \ y \in U, \rho(y,d) = \rho(x,d) \}. V(x,a)={ρ(y,a)∣ρ(y,a)isspecified,y∈U,ρ(y,d)=ρ(x,d)}.
